O Dev in Company

O Dev in Company surge para consolidar e suprir uma deficiência de comunicação entre empresas e profissionais do mercado, abrir portas para quem deseja levar conteúdo de qualidade para a comunidade e possibilitar uma maior interação e experiência entre os participantes, aumentando assim o network entre os presentes.

Quem vai estar lá?

Daniel Coscarelli

Stefanini

Daniel Coscarelli é UX Designer, e trabalha como designer web há mais de 10 anos, e 5 anos como designer mobile. Sempre criando uma interfaces conceituais, procura aplicar seus conhecimentos e técnicas visuais de Gestalt, semiótica, tipologia, concepção,usabilidade e acessibilidade entre outros, buscando sempre as melhores práticas e experiências centradas no usuário.

Entregáveis de UX

Nessa palestra será abordado alguns dos principais entregáveis de UX que utilizo em uma consultoria. Imersão com pesquisa em profundidade, benchmarking, brainstorming, personas, user stories, the bridge, mapa de conteúdo, definição de KPI's, protótipos.

Bruno Trecenti

ThoughtWorks

Desenvolvedor de software pragmático e praticante da arte Agile Development por mais de 6 anos no desenvolvimento de software.

Micro-serviços com Node.js

Nesta palestra será apresentado o mundo dos micro-serviços e como usar ferramentas e frameworks do node.js para criar, testar e organizar uma aplicação. Será abordado os aspectos de um micro-serviço, como organizá-los usando o framework web express.js e como testá-los usando ferramentas como Mocha, chai, sinon, nock, e supertest. Por fim será discutido os aspectos de implantação e escalabilidade dos serviços

Pedro Chaves

Planet Expat

Desenvolvedor de software há quase 10 anos no mundo web. Atualmente é um dos organizadores do PHPMG e sempre pensa em como a qualidade do código pode afetar a experiência do usuário.

Desbancando mitos sobre o PHP e o futuro da linguagem

PHP tem uma má fama de ser uma linguagem ruim, mas esta fama vem mudando graças à ótimas features adicionadas desde a versão 5.3 nos últimos anos. Nesta palestra será mostrado a grande evolução do PHP e o que vem por aí na versão 7 (programada para sair este ano), provando porque PHP ainda é uma linguagem relevante para o desenvolvimento web hoje em dia.

Paulo Pires

TIDY

Programador, as vezes palestrante, apreciador de cerveja e criador de mimimi. Adora hardware, open source e escrever código. Hoje eu trabalho na TIDY, uma empresa incrível com sede em Los Angeles que faz com que a vida das pessoas seja melhor e mais fácil.

We work remotely!!!

Vamos destrinchar o conceito de trabalho remoto e mostrar como se comportar, onde procurar emprego, formas de contratação, salário e falar muito mais sobre esse modelo que já se tornou estilo de vida para muita gente.

Lucas Mezêncio

Méliuz

Desenvolvedor há 8 anos. Especializado em backend. Um dos organizadores do PHPMG. Acredito que o compartilhamento de ideias e informação é o melhor caminho para o conhecimento. Movido a música boa.

Controle de Versão com Git e como melhorar seu Workflow com GitFlow

Desenvolver software sem controle de versão é suicídio. Se você não utiliza, você deve estar perdendo muito tempo. Na palestra será abordado a utilização do Git como sistema de controle de versão e apresentado o GitFlow, que é uma maneira de otimizar seu trabalho (principalmente em equipe), ensinando a dividir o desenvolvimento em features, bug fixes, gerar releases e tags dentro do controle de versão.

Rômulo Augusto Santos

ThoughtWorks

TDD para seres humanos

Talk introdutória sobre a importância da prática do Desenvolvimento orientado a testes, e de como ele influencia no design do código e facilita os fluxos de desenvolvimento.
